allowed-tools:
  - Read
  - Write

## Arguments

- 요약 대상: 다음 중 하나 이상의 형태로 전달된다.
  - **텍스트**: 사용자가 직접 입력한 텍스트
  - **파일 경로**: 로컬 파일 경로 (예: `./notes.md`, `/path/to/document.txt`)
  - **URL**: 웹 페이지 URL (WebFetch로 내용을 가져온다)
- `title` (선택): 요약 제목. 생략 시 내용에서 자동 추출한다.

## Rules

- 출력 언어는 입력 내용의 언어를 따른다. 한국어 입력이면 한국어로, 영어 입력이면 영어로 요약한다.
- 오늘 날짜: `date +%Y-%m-%d` 명령으로 확인한다.
- 반드시 **화면 출력**과 **마크다운 파일 저장** 두 가지를 모두 수행한다.
- **요약의 핵심 원칙: 분량을 줄이는 것이 목적이다.** 요약 결과는 원문 분량의 1/3 이하여야 한다. 원문을 다시 풀어쓰거나 부연하는 것은 요약이 아니다. 정보를 압축하여 읽는 시간을 줄여주는 것이 요약의 가치다.

## Steps

1. **입력 수집**
   - 텍스트가 직접 전달되었으면 그대로 사용한다.
   - 파일 경로가 주어졌으면 Read 도구로 파일 내용을 읽는다.
   - URL이 주어졌으면 WebFetch로 내용을 가져온다.
   - 여러 소스가 주어지면 모두 합쳐서 처리한다.

2. **요약 작성**
   - 원문에서 핵심만 추려서 짧게 압축한다.
   - 구조:
     - **한줄 요약**: 전체 내용을 한 문장으로 압축
     - **핵심 포인트**: 3~5개 불릿. 한 포인트당 한 문장. 가장 중요한 것만 남긴다.
     - **결정/액션** (원문에 있을 때만): 한 줄씩 간결하게 나열. 부연 설명 없이 핵심만.
   - 작성 원칙:
     - 장황한 설명, 배경, 경위는 과감히 생략한다.
     - "~에 대해 논의하였다", "~가 이루어졌다" 같은 메타 서술은 쓰지 않는다. 논의 결과만 적는다.
     - 하나의 포인트에 여러 내용을 합칠 수 있으면 합친다.
     - "상세 요약" 섹션은 만들지 않는다. 핵심 포인트로 충분하다.

3. **화면 출력**
   - 요약 결과를 대화에서 직접 보여준다.
   - 형식:
     ```
     ## {제목}

     {한줄 요약}

     - {포인트 1}
     - {포인트 2}
     - ...

     **결정 사항**
     - {결정 1}
     - {결정 2}

     **액션 아이템**
     - {담당}: {할 일} ({기한})
     ```

4. **마크다운 파일 저장**
   - 파일명: `summary-YYMMDD-{제목 슬러그}.md`
     - 날짜는 오늘 날짜 기준, 2자리 연도
     - 제목 슬러그는 제목에서 공백을 `-`로 치환하고, 특수문자를 제거한 소문자 영문/한글 (최대 30자)
     - 예: `summary-260320-api-설계-회의.md`
   - 스킬을 실행한 현재 작업 디렉토리에 저장한다.
   - 파일 내용은 화면 출력과 동일하되, 상단에 메타 정보를 추가한다:
     ```
     > 원본: {파일명 또는 URL 또는 "직접 입력"}
     > 생성일: {YYYY-MM-DD}
     ```
   - 저장 후 파일 경로를 사용자에게 알려준다.
