# Worklog Generator

TickTick에서 완료된 작업을 가져와서, 날짜별로 정리된 worklog를 생성하는 스킬입니다.

## 입력

사용자가 날짜 기간을 지정하지 않았거나 인자 없이 스킬만 호출한 경우, 아래 가이드를 출력하고 입력을 기다린다:

```
Worklog 생성기 - 사용법:
  /worklog 3/17 ~ 3/21    특정 기간
  /worklog 이번주          이번 주 (월~금)
  /worklog 지난주          지난 주 (월~금)
  /worklog 3월             3월 전체
  /worklog 3/17            특정 날짜 하루
```

날짜 기간이 주어진 경우, 다양한 형태로 올 수 있습니다:
- "3/17 ~ 3/21" → 2026-03-17 ~ 2026-03-21
- "이번주" → 이번 주 월~금
- "지난주" → 지난 주 월~금
- "3월" → 3월 1일 ~ 3월 말일
- 날짜 하나만 → 해당 날짜 하루

상대적 표현은 오늘 날짜를 기준으로 절대 날짜로 변환하세요.

## 주말 처리

- 기본적으로 주말(토, 일)은 **제외**합니다.
- 사용자가 "주말 포함", "토요일도" 등 명시적으로 요청한 경우에만 주말을 포함합니다.

## 절차

### 1단계: 완료된 작업 가져오기

`mcp__ticktick__list_completed_tasks_by_date`를 사용하여 해당 기간의 완료된 작업을 조회합니다.

TickTick API는 최대 14일까지만 한 번에 조회할 수 있으므로, 기간이 14일을 넘으면 여러 번 나눠서 호출하세요.

```
search:
  startDate: "2026-03-17T00:00:00+09:00"
  endDate: "2026-03-21T23:59:59+09:00"
```

### 2단계: 날짜별로 작업 분류

조회된 작업들을 `completedTime` 기준으로 날짜별 그룹으로 나눕니다. 주말은 기본 제외.

### 3단계: 하루 작업 요약 (최대 4개)

각 날짜에 완료된 작업들을 내용의 유사성과 프로젝트 기준으로 묶어서 **최대 4개의 작업 항목**으로 요약합니다.

요약할 때의 원칙:
- 비슷한 성격의 작업은 하나로 묶어서 쉼표로 연결 (예: "ABC 업데이트, XYZ 수정")
- 작업 제목이 이미 명확하면 그대로 사용
- 지나치게 세부적인 내용은 상위 개념으로 추상화
- 해당 날짜에 완료된 작업이 없거나 매우 적으면, 있는 만큼만 표시 (억지로 4개를 채우지 않음)

### 4단계: 8시간을 30분 단위로 분배

하루 총 **8시간(= 16 슬롯 x 30분)**을 요약된 작업들에 분배합니다.

분배 규칙:
- 단위는 30분 (0.5h). 표시 형식은 `[Xh]` 또는 `[X.5h]`
- 모든 작업 시간의 합은 반드시 **8h**
- 작업량과 복잡도를 고려해서 비율을 적절히 조절
- 최소 단위는 0.5h (30분). 작업이 아무리 작아도 0.5h 이상
- 작업이 1~2개뿐이라도 합계는 8h를 유지

시간 표기:
- 정수 시간: `[1h]`, `[2h]`, `[3h]`
- 반시간: `[0.5h]`, `[1.5h]`, `[2.5h]`

### 5단계: 출력

아래 형식으로 출력합니다:

```
2026-03-17 (월)
[3h] 사용자 인증 모듈 리팩토링
[2.5h] API 엔드포인트 버그 수정, 에러 핸들링 개선
[1.5h] 배포 스크립트 업데이트
[1h] 코드 리뷰, PR 머지

2026-03-18 (화)
[4h] 결제 시스템 연동 개발
[2h] 단위 테스트 작성
[1.5h] DB 마이그레이션 스크립트 작성
[0.5h] 팀 미팅, 이슈 트리아지
```

출력 규칙:
- 날짜 옆에 요일을 괄호로 표시: `(월)`, `(화)`, ...
- 시간이 큰 작업부터 위에 배치 (내림차순)
- 날짜 사이에 빈 줄 하나
- 추가 설명이나 머리말/꼬리말 없이 worklog 본문만 출력

## 엣지 케이스

- **완료된 작업이 없는 날**: 해당 날짜를 건너뜁니다. 빈 날은 출력하지 않습니다.
- **작업이 1개뿐인 날**: `[8h] 작업내용` 한 줄로 출력합니다.
- **기간 내 평일이 하나도 없는 경우**: "해당 기간에 평일이 없습니다"라고 안내합니다.
- **TickTick에 데이터가 전혀 없는 경우**: "해당 기간에 완료된 작업이 없습니다"라고 안내합니다.
